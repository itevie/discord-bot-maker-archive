Hosting a bot on an external host

Of course, you can do this normally by just copying the bot's data to another computer and using the CLI, but this way is a lot more convienient for you.

Getting started

To get started, install Discord Bot Maker on whichever computer you want to (make sure you install the right OS version too)

Next test if it has installed correctly by running

(appname) --version

But replacing (appname) with your app's name:

botMaker.exe --version

Creating the server

This can be done in 1 command:

(appname) --server=(port)

But replacing (appname) with the app's name and (port) with the port you want the server to run on (this does not matter much):

botMaker.exe --server=8080

Information

The versions of the server and the current app your running needs to be the same, otherwise the server will refuse to run any bots you give it.

Connecting to your server

Next, you need to find the IP of the computer to connect to, botMaker cannot get this so you need to find it yourself.

Don't know how? Either search on the internet: "How to get IP address of (OS name)", or use a service like NGROK and run:

ngrok http (port you used)

Next, copy down the IP and port you got, go to the web tab in Discord Bot Maker and write the IP and port into the appropiate text boxes, run "Check" and your done!

Starting bots on my server

To start a bot on your server, all you need to do is go to the "Bots" tab, and click the globe icon next to the bot you want to start.

Troubleshooting

Error: connect ECONNREFUSED (url)

This means that the app failed to connect to the server, solutions:

  • Start the server
  • Check the URL and port are correct
  • Navigate to the server settings on the sidebar and select "stop trying to connect"

If above fail, please report the error
