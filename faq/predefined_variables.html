How predefined variables work

First of all, these variables are all enclosed in {{ }} just like your own variables, they can be used in any input in the command creator

Predefined variables are variables that are automatically created in certain circumstances

Arguments

These are parts of a command, for example: !ban @john hello the arguments (or args) in this example are @john and "hello"

Each arg has its own variable assigned: {{argn}} where n is the index in which it appears

Example

!ban @john "spamming too much"

The variables that will be assigned in the following example are:

  1. {{arg1}} - John's User ID
  2. {{arg2}} - "Spamming too much"

After Args

These work similar to args, but they give everything after a certain argument, for example:

If I ran: !ban @john Not enough fun

And my code was:
ban {{arg1}} {{afterarg1}}

{{arg1}} would be "@john" and {{afterarg1}} would be "Not enough fun"

Information

The argument types are ONLY defined when the command type uses a prefix, not if it just checks if the message contains some string.

Message Variables

All of these variables start with message: for example: {{message:id}}

A list of common ones are:

  • {{message:id}}
  • {{message:content}}
  • {{message:reference:messageId}} - If the message replies to another message it will give that message's ID
  • {{message:author:id}} - The ID of the person who created the message

Result Variable

This is a crucial variable to be able to do mostly anything, it is accessed using {{result}} anywhere

It is set to the result of the previous ran action. For example:

add 2 2

There is absolutely no way to access the result from the above except {{result}}:

add 2 2
set-variable x {{result}}
message-author-reply {{x}}

This would work, obviously in the action UI form

Misc variables

  • {{uptime}} - Your bot's uptime in milliseconds
  • {{prefix}} - The bot's prefix
  • {{ping}} - The bot's ping
  • {{api_ping}} - The bot's ping to discord.com
