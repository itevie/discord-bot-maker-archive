How to use the Database

The databse is used to store bot-wide variables that can be used at any time and are saved permanently

Unlike command variables, these never auto-delete after it is done. Unlike variables defined by you, they can be changed and deleted within a command's actions.

Using the Database UI

In the Database page, there is a couple buttons to add new ones, search etc.
There are also a list of all database keys below.

Using the Database within commands

You can get values, change values and delete keys within the command's actions

Fetching a value

To fetch a value in your database, create a new action and set it to "Fetch Database Item" and in the ID section, put in the name of the key

After this, it will be set to the {{answer}} variable for you to use later

You can also use {{db:key}} anywhere to, where key is the key name

Updating a value

To update a value, you must create a new action and set it to "Update Database Item", put the ID section as the key name, and the content section as to what to set it to.

Deleting a value

To delete a value, you must create a new action and set it to "Delete Databsae Item", put the ID section as the key name and you're done.

Naming Convention

As this database does not have subkeys, you must work around this.

For saving data in guilds, I recommend the following scheme: Guild ID_Settings, a few examples are:

  1. db:972599308966846506_prefix - The guild's prefix
  2. db:972599308966846506_users_957418858866683925_xp - A user in the guild's XP
  3. db:{{message:guildId}}_users_{{message:author:id}}_xp - Same as above but the IDs depend on who ran it

Obviously, you do not have to follow this, though it is recommended.
